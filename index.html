Worked example — Triangle + tail

Walk-through on a small graph.
▶

Consider V = {A, B, C, D} and E = {A-B, A-C, B-C, C-D}. A triangle {A,B,C} plus a pendant D attached to C.

  1. All vertices uncolored at the start.
  2. Process A — no colored neighbors, assign Color 1.
  3. Process B — neighbor A has 1, assign Color 2.
  4. Process C — neighbors A(1), B(2) exclude 1 & 2, assign Color 3.
  5. Process D — only C(3) is excluded, assign Color 1.

Result: valid 3-coloring, exactly 3 colors.

Problem definition

The 3-coloring decision problem in precise terms.
▶

A proper k-coloring of a graph G = (V, E) is an assignment c : V → {1, …, k} such that for every edge (u, v) ∈ E, c(u) ≠ c(v). The 3-Coloring problem asks: given an undirected graph G, does a proper 3-coloring exist?

What does "at most 3 colors" mean?

It means you may use 1, 2, or 3 distinct color labels across all vertices, and every edge must connect two different labels. A 2-colorable graph is automatically 3-colorable (use two of the three available colors).

Why 2-coloring is easy

Bipartiteness vs. 3-chromaticity.
▶

A graph is 2-colorable if and only if it is bipartite — its vertex set can be split into two independent sets. A simple BFS or DFS assigns alternating labels and checks for conflicts, giving a O(V + E) algorithm.

Adding just one more color to the palette changes the computational landscape completely: the decision problem for 3 colors is NP-complete. The boundary between 2 and 3 colors is the classical boundary between easy (polynomial) and intractable in graph coloring.

Greedy vs Backtracking — paradigms

The two algorithms this visualizer implements.
▶

Greedy paradigm

At each step, assign the current vertex the smallest color index that none of its already-colored neighbors uses. Runs in O(V + E). Simple and fast, but not guaranteed to be optimal — the ordering of vertices determines how many colors it ultimately uses.

Backtracking paradigm

Try color 1 for the next vertex; if it fits, recurse. On failure, undo and try the next color. Explore the full assignment space. Always correct; worst case O(3ⁿ), so it only scales to small graphs.

Why ordering affects greedy

Greedy's output depends on the order. Visit a "bad" order on an otherwise 3-colorable graph and greedy may still need a 4th color. Try the Greedy trap preset with Original ordering then again with Desc. degree.

NP-Completeness of 3-Coloring

Membership in NP, and reduction from 3-SAT.
▶

1. 3-Coloring is in NP

To show membership in NP, we provide a polynomial-time verifier. Given a graph G and a candidate coloring c (the certificate), the verifier checks in O(|V| + |E|):

  • each vertex receives a label in {1, 2, 3}, and
  • no edge (u, v) has c(u) = c(v).

2. Certificate & Verifier

Certificate c : V → {1,2,3} Verifier(G, c): for each v in V: check c(v) ∈ {1,2,3} for each (u,v) in E: check c(u) ≠ c(v) return all checks passed

3. Reduction from 3-SAT

We show 3-SAT ≤p 3-COLOR. From a 3-SAT formula φ = C1 ∧ … ∧ Cm over variables x1, …, xn we construct a graph Gφ such that φ is satisfiable iff Gφ is 3-colorable. The three color classes encode True (T), False (F), and Base (B).

4. Truth gadget (T/F/B triangle)

Add three special vertices T, F, B, pairwise connected into a triangle. Any 3-coloring assigns them three different colors; relabel so those colors are called True, False, and Base.

5. Variable gadget

For each variable xi add literal vertices vi and ¬vi. Connect vi—¬vi, and connect both to B. This forces them to take the two non-Base colors — one gets T, the other F — giving a consistent truth value per variable.

6. Clause gadget (OR-gadget)

For each clause ℓ1 ∨ ℓ2 ∨ ℓ3, attach a bounded-size gadget built from three 2-input OR sub-gadgets. Its output is constrained (via connection to F and B) to be colored T, which is only possible when at least one of the literals is colored T.

7. Forward (φ satisfiable ⇒ Gφ is 3-colorable)

Given a satisfying assignment, color literal vertices T/F accordingly. Each clause has at least one true literal, so every OR-gadget can be completed consistently, yielding a valid 3-coloring of the whole graph.

8. Backward (Gφ 3-colorable ⇒ φ satisfiable)

Given a 3-coloring, the triangle fixes meanings for T, F, B. The variable gadget forces each vi / ¬vi pair to split between T and F, defining an assignment. The OR-gadget's output being colored T forces at least one input to be T, so every clause is satisfied.

9. Polynomial-time construction

|V(Gφ)| = O(n + m), |E(Gφ)| = O(n + m), and the whole graph is built in polynomial time. Hence 3-SAT reduces to 3-COLOR in polynomial time.

10. Conclusion

  • 3-COLOR ∈ NP,
  • 3-SAT ≤p 3-COLOR,
  • 3-SAT is NP-hard ⇒ 3-COLOR is NP-complete.
  • No polynomial-time algorithm is known — and assuming P ≠ NP, none exists.

Trade-off cheat sheet

When to reach for which approach.
Use Greedy when…
the graph is large and an approximate coloring is acceptable. Fast, deterministic per ordering, but may overuse colors.
Use Backtracking when…
you need to decide exactly whether a 3-coloring exists on a small/medium graph. Correct but exponential in the worst case.
Greedy limitation
it commits to a color per vertex and never reconsiders. A poor ordering can force an extra color.
Backtracking strength
exhaustive: if it returns "no", no 3-coloring exists. If it returns "yes", the coloring is valid.

Gadget explanation — a closer look

Each gadget's coloring constraint, in isolation.
Truth Gadget
  • The triangle forces 3 distinct colors — one per vertex.
  • T represents True.
  • F represents False.
  • Base is a neutral third color, used as a reference.
Variable Gadget
  • x and ¬x must take opposite truth colors.
  • Both are joined to Base, so neither can be colored Base.
  • Whichever is True, the other is forced to False — the gadget itself doesn't decide which.
Clause Gadget (OR)
  • The gadget behaves like an OR condition on its three inputs.
  • Output can be colored True only when at least one literal is True.
  • If every literal were False, the gadget cannot be 3-colored — the clause is unsatisfied.

FFT ✓ valid

Why the reduction works

The three facts that make 3-Coloring NP-complete.
✓ In NP
A coloring is a certificate. The verifier reads the proposed color for each vertex and checks every edge in polynomial time.
→ Forward direction
If the formula is satisfiable, assign colors according to the truth assignment, and every gadget can be coloured consistently.
← Backward direction
If the graph is 3-colorable, the gadget colors define a satisfying assignment for the formula — each clause has at least one True literal.
Polynomial-time construction. The construction is polynomial because we add only a constant-size gadget for each variable and each clause. Hence |V(Gφ)| = O(n + m) and |E(Gφ)| = O(n + m).

Greedy coloring is fast because it checks each vertex and its edges once, so its work is proportional to V + E. Backtracking is exact, but in the worst case it may try up to 3ⁿ color assignments. This is why exact solving becomes impractical as n grows. Note that greedy is not guaranteed to find the minimum number of colors — its result can depend on the vertex ordering.

Practical guidance

When to reach for which method.
⚠ Exponential worst case
Backtracking is useful for small graphs because it gives an exact answer — either a valid 3-coloring or a proof that no 3-coloring exists. For larger graphs, the exponential worst case becomes too expensive: the search space can reach up to 3ⁿ color assignments. Good implementations prune heavily, but the worst case remains exponential.

Why the exponential blow-up?

Backtracking on 3-coloring explores an implicit search tree: at each of the n vertices, up to 3 colors are attempted. In the worst case this tree has up to 3ⁿ leaves. Real implementations prune aggressively (infeasible branches abort early), but the worst case remains exponential.

Greedy, by contrast, commits one color per vertex and never revisits. Its work is linear in the input size, O(V + E). The price: it can produce non-optimal colorings, and will sometimes use 4+ colors on a graph that is demonstrably 3-colorable.
